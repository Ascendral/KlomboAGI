# KlomboAGI — Definition

## North Star

> An autonomous system that completes unseen digital-work tasks across domains,
> with low intervention, measurable learning, and transfer.

## Hard Gates

Every claim must pass ALL of these:

1. **Unseen tasks only** — No task in the eval suite was used during development
2. **Multi-domain** — Coding, research, writing, data analysis, ops — minimum 3 domains
3. **Longitudinal improvement** — Performance must measurably improve over repeated episodes
4. **Third-party reproducibility** — An outsider can clone the repo and reproduce the results
5. **No fixture-shaped intelligence** — No hardcoded heuristics that only work on demo tasks

## What AGI Means Here

We do NOT use "AGI" to mean:
- Superhuman intelligence
- Consciousness
- General knowledge of everything

We DO use "AGI" to mean:
- **General**: works across task types it wasn't specifically built for
- **Autonomous**: completes tasks with minimal human intervention
- **Learning**: gets better from experience, not just from code changes
- **Transferable**: knowledge from domain A helps in domain B

## Stages

| Stage | Name | Gate |
|-------|------|------|
| 0 | Current | Strategy search + pattern matching (100 ARC puzzles) |
| 1 | General task executor | Completes unseen coding + research tasks with tools |
| 2 | Memory that matters | Retrieved memory measurably improves future success |
| 3 | Transfer | Success on new repos/domains without hand-tuned heuristics |
| 4 | Long-horizon autonomy | Runs for hours, recovers from failure, resumes |
| 5 | External proof | Blind evals, outside replication, strong baselines |

## When We Call It AGI

Only when:
- Stage 5 gates are passed
- Eval curves make the argument (not marketing)
- Outside people reproduce it
- We can't find a way to game the evals
